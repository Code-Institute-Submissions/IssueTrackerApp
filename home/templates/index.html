IDHub

Overview

Mobile application is used in a couple of scenarios, covering everything from Lite KYC to Enhanced Due Diligence process, as required by the end system risk level and compliance. Data is being acquired and verified based on the KYC profile.


Basic, Lite KYC

Intended to be used for some absolute low risk customers for non-financial services, providing customer unique identification.


Full KYC

Intended to be used for regular customers for non-financial services or financial services with a later face-to-face verification.


Enhanced KYC

Intended to be used for customers using financial services.


The solution is API based and uses 3rd party services for mobile number verification, login and videoconferencing.

Mobile Application

  • for Android

Components used in design:

  • SQL server database
  • oAuth 2.0 based universal login system (internal development) with Firebase based authentication
  • 3rd party videoconferencing solution
  • Mobile Application
  • API Service with business logic and database access
